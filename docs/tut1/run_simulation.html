3.4. Run simulation

Models stored in LM-format files can be executed by Lattice Microbes (LM). LM is not a Python module, but is executed by the command prompt:

$ mkdir results_photobleach
$ cp models/photobleach.lm results_photobleach/0000.lm
$ lm -r 1 -sp -sl lm::rdme::MpdRdmeSolver -f results_photobleach/0000.lm

The above commands make a copy of ‘models/photobleach.lm’ as ‘results_photobleach/0000.lm’, and it is executed by LM. This process can also be described using a Python script, as follow (41_single_run.py):

41_single_run.py
 1import os, sys, shutil
 2import subprocess as s
 3
 4filename_lm     = 'models/photobleach.lm'
 5filename_run    = 'results_photobleach/0000.lm'
 6
 7if os.path.isfile(filename_run):
 8    os.remove(filename_run)
 9    print('Previous run file was removed.')
10
11print('Create a run file: ', filename_run)
12os.makedirs('results', exist_ok=True)
13shutil.copy(filename_lm, filename_run)
14
15com = ['lm','-r', '1', '-sp', '-sl','lm::rdme::MpdRdmeSolver','-f', filename_run]
16print(' '.join(com))
17s.call(com)

We can easily run simulation; however, in this simple run, we cannot handle any event functions, such as photobleaching and synaptic input. The “ConnectRun” class enables them by connecting multiple runs into a sequential run. Events can be inserted in between the unit runs. The ‘42_connect_run.py’ is an example:

42_connect_run.py
 1import sys, os
 2import numpy as np
 3from pyLD import *
 4
 5
 6def event_replace(lattice, sys_param, event_param):
 7	i    = sys_param['i']
 8	time = sys_param['time']
 9	s    = sys_param['species']
10	src  = s[event_param['src']]
11	dst  = s[event_param['dst']]
12	print('\nEvent at: {:g}, Current time: {:.3f}\n'.format(i, time))
13
14	labeled_area = (sys_param['label volume'] == 1 )
15	for i in range(lattice.shape[3]):
16		t_lattice = lattice[:,:,:,i]
17		t_lattice[labeled_area*(t_lattice == src)] = dst
18	return lattice, event_param
19
20
21r = ConnectRun()
22r.label_volume_file = 'models/labels_ball_and_stick.h5'
23
24# FRAP
25r.template_lm_file  = 'models/photobleach.lm'
26r.output_dir        = 'results_photobleach'
27r.event_params      = {'src':'YFP', 'dst':'bleached YFP'}
28r.exec_periods      = [4.0, 4.0]
29r.exec_events       = [event_null, event_replace]
30
31'''
32# Ca2+ influx via NMDARs
33r.template_lm_file  = 'models/Ca_dynamics.lm'
34r.output_dir        = 'results_Ca_dynamics'
35r.event_params      = {'src':'inactive NMDAR', 'dst':'active NMDAR'}
36r.exec_periods      = [2.0, 2.0]
37r.exec_events       = [event_null, event_replace]
38'''
39
40'''
41# Multiple GPUs
42GPU_ids  = '0,1'
43num_GPUs = '2'
44num_CPUs = '2'
45r.lm_option   = ['-g', GPU_ids, '-gr', num_GPUs,'-cr', num_CPUs, '-sl','lm::rdme::MGPUMpdRdmeSolver']
46'''
47
48r.exec()

That is all for simulation run.
